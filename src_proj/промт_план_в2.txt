**Цель:** Создать детализированный пошаговый план разработки проекта на основе предоставленной архитектуры.  

---

## **Задача**  
На основе контекста сформируй **детальный план разработки** (ТЗ).

**Требования к плану:**  
1. **Структура:**  
   - Разделение на три основные роли:  
     - **ML + База данных** (подготовка модели, настройка MongoDB).  
     - **Бэкенд** (API, бизнес-логика, интеграция с ML и БД).  
     - **Фронтенд** (UI/UX, интеграция с API).  
   - Для каждого этапа укажи:  
     - Конкретные задачи (например: "Настроить схемы MongoDB для сущностей Document и Summary").  
     - Зависимости от других этапов (например: "Требует завершения этапа ML").  
     - Критерии завершения (например: "Работоспособный интерфейс", "Корректные связи в БД для таблицы Documents").  
     - Приоритет (высокий/средний/низкий).  

2. **Детализация:**  
   - Укажи порядок разработки (что должно быть сделано в первую очередь).  
   - Упомяни кросс-функциональные задачи.  
   - Добавь измеримые критерии успеха.  

3. **Ограничения:** 
   - Сохраняй фокус на архитектурных решениях, а не на коде.  

---

## **Пример ожидаемого формата плана**  
```
Этап 1: [Название этапа]  
Роль: [ML + БД / Бэкенд / Фронтенд]  
Задачи:  
- [Конкретная задача 1]  
- [Конкретная задача 2]  
Зависимости: [Этапы или компоненты, от которых зависит задача]  
Критерии завершения: [Измеримые результаты]  
Приоритет: [Высокий/Средний/Низкий]  

Этап 2: [...]  
...  
```  

--- 

**Важно:** План должен быть реализуемым в условиях реальной разработки. Учитывай ограничения (15 МБ на файл, поддержка только русского языка).
